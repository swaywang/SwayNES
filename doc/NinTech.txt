For information about 6502 instructions,</div><div class='line' id='LC150'>	addressing and opcodes or cycle by cycle steps, Look for 6502.txt, 6502_cpu.txt at nesdev</div><div class='line' id='LC151'>	(http://nesdev.parodius.com/ or http://nesdev.com/) and drop by http://www.6502.org/</div><div class='line' id='LC152'><br/></div><div class='line' id='LC153'>	For technical information about the 2A03 look for Brad Taylor&#39;s excellent document called &quot;2A03</div><div class='line' id='LC154'>	technical reference&quot; which covers absolutely everything about the chip. The speed of the CPU on NTSC</div><div class='line' id='LC155'>	is 1.7897725 MHz and 1.6626070 MHz for PAL. The CPU run exactly 341/3(NTSC) and 341/3.2 (PAL) cycles</div><div class='line' id='LC156'>	per scanline. Below is some general information to help you understand the 6502 better.</div><div class='line' id='LC157'><br/></div><div class='line' id='LC158'>2c Interrupts</div><div class='line' id='LC159'>-------------</div><div class='line' id='LC160'><br/></div><div class='line' id='LC161'>	The 6502 has 3 in Interrupts each of them jump to an address based on their vectors.</div><div class='line' id='LC162'>	The address is 2 bytes long with the low byte first.</div><div class='line' id='LC163'><br/></div><div class='line' id='LC164'>	$FFFA: NMI</div><div class='line' id='LC165'>	$FFFC: RESET</div><div class='line' id='LC166'>	$FFFE: IRQ/BRK</div><div class='line' id='LC167'><br/></div><div class='line' id='LC168'>	Non-Maskable Interruption (NMI):</div><div class='line' id='LC169'>	      Pushes Flags, set I flag then jump, 7 cycles long.</div><div class='line' id='LC170'>	      It is triggered when reg $2000.7 is set when the PPU hits vblank.</div><div class='line' id='LC171'>	      If you clear $2000.7 then set it while $2002.7 is still set, (look at PPU regs) you can trigger</div><div class='line' id='LC172'>	      another NMI. NMI is used to jump to code that updates the screen or time somthing.</div><div class='line' id='LC173'><br/></div><div class='line' id='LC174'>	Reset:</div><div class='line' id='LC175'>	      Jumps to Address. This happens when you hit the reset button on the NES hardware, or when you</div><div class='line' id='LC176'>	      turn on your NES. This can also happen if you run a bad opcode, not all-bad opcodes will cause a</div><div class='line' id='LC177'>	      reset but they may cause one. You should wait 2 VBlanks so everything can get in sync.</div><div class='line' id='LC178'><br/></div><div class='line' id='LC179'>	Interrupt ReQuest (IRQ):</div><div class='line' id='LC180'>	      if the I flag is cleared, IRQ pushes the Flags, set the I flag and spends 7 cycles doing it.</div><div class='line' id='LC181'>	      If the I flag is already set nothing will happen. Based on what is causing the IRQ it may be</div><div class='line' id='LC182'>	      fired regularly until it is acknowledged (usually in the IRQ code).</div><div class='line' id='LC183'><br/></div><div class='line' id='LC184'>	      It is usually timed for mid frame tricks or to regulate something such as sound.</div><div class='line' id='LC185'><br/></div><div class='line' id='LC186'>	Break (BRK):</div><div class='line' id='LC187'>	      Pushes Flags with the B flag set. Then sets the I flag, it is 7 cycles long.</div><div class='line' id='LC188'>	      This is caused when the 6502 runs the instruction BRK which is 2 bytes long.</div><div class='line' id='LC189'>	      The 2nd byte of BRK is not used in any way, usually set a specific value for debugging</div><div class='line' id='LC190'>	      purposes. This Instruction shares vectors with IRQ and is mostly used for debugging.</div><div class='line' id='LC191'><br/></div><div class='line' id='LC192'>	      NOTE: The B flag does not exist in the 6502 Status flags. The bit is only pushed, when read in</div><div class='line' id='LC193'>	      It is cleared.</div><div class='line' id='LC194'><br/></div><div class='line' id='LC195'>2d Registers</div><div class='line' id='LC196'>------------</div><div class='line' id='LC197'><br/></div><div class='line' id='LC198'>DMA (Direct Memory Access, $4014):</div><div class='line' id='LC199'>	The DMA is activated when a value is written to $4014.
